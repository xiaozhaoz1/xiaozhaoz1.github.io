Docker

介绍

Docker 是一个开源的应用容器引擎，基于 Go 语言 并遵从 Apache2.0 协议开源。

Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。

容器是完全使用沙箱机制，相互之间不会有任何接口（类似 iPhone 的 app）,更重要的是容器性能开销极低。

容器 vs. 虚拟机

Docker 的核心是 容器化技术。为了更好地理解容器，我们将其与虚拟机对比。

虚拟机 (Virtual Machine):

虚拟机在物理硬件上运行一个完整的客户操作系统（Guest OS）。这个 Guest OS 通过一个叫做管理程序 (Hypervisor) 的中间层（如 VMware, VirtualBox）来访问底层硬件。

缺点: 每个 VM 都包含一整个 OS，体积庞大（通常几 GB 到几十 GB）、启动缓慢、并且消耗大量系统资源（CPU、内存）。

容器 (Container):

容器与主机共享同一个操作系统内核（Linux Kernel）。容器中只包含应用程序及其依赖的库、二进制文件和配置文件。

它们通过 Docker 引擎 (Engine) 运行在主机操作系统上，引擎负责资源的隔离和分配。

优点: 极其轻量（通常只有 MB 级别）、启动迅速（秒级）、资源开销极低。你可以在一台主机上运行成百上千个容器。

简单比喻:

虚拟机就像买了一整栋新房子（完整的操作系统），只是为了在里面放一张床（你的应用）。

容器就像在一个大型公寓楼（主机操作系统）里租了一个单间。所有房间共享地基和基础设施（内核），但每个房间都是独立、隔离的。

Docker 的核心概念

镜像 (Image):

一个 只读的模板，类似于虚拟机的快照（Snapshot）或安装光盘。

它包含了运行某个软件所需的所有内容：代码、运行时环境、库、环境变量和配置文件。

镜像本身是静态的、不可改变的。你可以用它来创建一个或多个容器。

例如: nginx:latest 镜像包含了在 Linux 上运行 Nginx 所需的一切。

容器 (Container):

镜像是模板，容器就是根据这个模板创建和运行的实例。

容器是镜像的运行态。你可以启动、开始、停止、删除容器。

容器与镜像的关系，就像面向对象编程中 对象与类 的关系。

容器是轻量级和可隔离的，每个容器都运行在独立的环境中，互不干扰。

仓库 (Registry):

一个集中存储和分发镜像的地方，最著名的是 Docker Hub (https://hub.docker.com)，类似于代码仓库 GitHub。

你可以从仓库 拉取 (pull) 别人构建好的镜像（如 Redis, MySQL, Node.js），也可以将自己构建的镜像 推送 (push) 到仓库（公开或私有）以供他人使用。

Docker 的应用场景

微服务架构：每个服务独立容器化，便于管理和扩展。

CI/CD流水线：与 Jenkins/GitLab CI 集成，实现自动化构建和测试。

开发环境标准化：新成员一键启动全套依赖服务（如数据库、消息队列）。

云原生基础：Kubernetes 等编排工具基于 Docker 管理容器集群。

安装与配置 Docker

在 Windows 上安装 (Windows 10/11)

前提条件:

开启 BIOS 中的虚拟化支持（VT-x）。通常开机按 F2/Del 键进入 BIOS 设置。

对于 Windows 10 以下版本，需要安装 Docker Toolbox。但对于 Win10/Win11，请使用 Docker Desktop。

步骤:

访问 Docker 官网下载页面: https://www.docker.com/products/docker-desktop

下载 Windows 版的 Docker Desktop Installer.exe。

双击运行安装程序。安装过程中会要求你启用 Windows 的 WSL 2 (Windows Subsystem for Linux 2) 功能。

安装完成后，重启电脑。

在开始菜单中找到并运行 “Docker Desktop”。

打开 命令提示符 (CMD) 或 PowerShell，输入 docker --version 和 docker run hello-world 来测试安装是否成功。

在 macOS 上安装

步骤:

访问 Docker 官网下载页面: https://www.docker.com/products/docker-desktop

下载 Apple Chip (M1/M2) 或 Intel Chip 版本的 Docker Desktop.dmg。

双击打开下载的 .dmg 文件，将 Docker 的鲸鱼图标拖拽到 Applications 文件夹中。

在启动台 (Launchpad) 中找到并运行 “Docker Desktop”。

首次运行会请求一些系统权限，请同意授权。

等待顶部菜单栏出现 Docker 的鲸鱼图标，表示已启动。

打开 终端 (Terminal)，输入 docker --version 和 docker run hello-world 来测试安装是否成功。

安装成功后，让我们通过 hello-world 镜像来熟悉最基本的 Docker 命令流程。

寻找镜像: Docker 首先会在本地查找是否有 hello-world 这个镜像。

拉取镜像: 如果本地没有，Docker 会自动去 Docker Hub 这个默认仓库下载（拉取）它。

创建并运行容器: 下载完成后，Docker 会根据这个镜像创建一个容器并运行它。

容器内的程序很简单：就是打印出一段 “Hello from Docker!” 的欢迎信息和一些说明文字，然后立即停止。

常用的命令

docker --version: 查看 Docker 版本信息，确认安装成功。

docker info: 显示更详细的 Docker 系统信息，包括容器和镜像的数量、存储驱动等。

docker pull <image_name:tag>: 拉取镜像。tag 通常是版本号，如 ubuntu:20.04。如果不指定 tag，默认是 latest。

docker images: 列出本地所有镜像。你会看到刚才拉取的 ubuntu:20.04 和 hello-world。

docker run <image_name> : 创建并运行一个容器。这是最核心的命令。

-it: 这是两个选项的组合。-i 表示交互式操作，-t 表示分配一个伪终端。通常一起使用，以便我们与容器进行交互，就像打开了一个 Shell 一样。

--name: 为容器指定一个名字，而不是使用随机生成的名字。

-d: 让容器在后台运行（守护态）。

-p host_port:container_port: 映射端口，将主机的端口映射到容器的端口。

-v host_dir:container_dir: 挂载数据卷，将主机上的目录挂载到容器中，实现数据持久化。

日常管理命令

容器生命周期管理

docker ps: 列出正在运行的容器。

docker ps -a: 列出所有容器（包括已停止的）。

docker stop <container_name_or_id>: 停止一个运行中的容器。

docker start <container_name_or_id>: 启动一个已停止的容器。

docker restart <container_name_or_id>: 重启容器。

docker rm <container_name_or_id>: 删除一个已停止的容器。加 -f 选项可以强制删除运行中的容器（不推荐）。

docker container prune: 删除所有已停止的容器，用于清理空间。

查看容器信息与日志

docker logs <container_name_or_id>: 查看容器的日志输出。加 -f 选项可以持续跟踪日志（类似 tail -f）。

docker inspect <container_name_or_id>: 获取容器（或镜像）的底层详细信息（元数据），以 JSON 格式返回。包括IP地址、端口映射、挂载卷等所有信息。

docker exec -it <container_name_or_id> : 在正在运行的容器中执行命令。这是非常强大的调试工具。

使用 exec 进入容器后，输入 exit 不会导致容器停止，只会退出当前的 exec 会话。

镜像管理

docker images: 列出本地镜像。

docker rmi <image_id>: 删除本地镜像。如果镜像有容器正在使用（即使已停止），需要先删除容器才能删除镜像。

docker search : 在 Docker Hub 中搜索镜像。

docker pull <image_name:tag>: 拉取镜像，如前所述。

docker image prune: 清理未被使用的镜像（悬虚镜像）。

逐行解释:

FROM: 指定基础镜像。我们基于轻量级的 python:3.8-slim 开始构建。

WORKDIR: 设置工作目录，相当于 cd /app。

COPY: 将主机上的文件复制到镜像中。分两步复制是为了利用 Docker 的构建缓存。先复制依赖文件并安装依赖，这层缓存只有在 requirements.txt 改变时才会失效。最后再复制代码，这样修改代码时不需要重新安装依赖，大大加快构建速度。

RUN: 在构建镜像时执行命令。这里用于安装 Python 依赖。

EXPOSE: 声明容器运行时监听的端口，这是一个元数据，方便他人理解。

ENV: 设置环境变量。

CMD: 指定容器启动时默认运行的命令。一个 Dockerfile 中只能有一条 CMD 指令。

数据持久化与卷 (Volumes)

容器默认的文件系统是临时的。当容器被删除时，其中的所有更改也会丢失。为了解决这个问题，Docker 提供了 卷 (Volumes) 和 绑定挂载 (Bind Mounts)。

卷 (Volumes)

卷是由 Docker 完全管理的持久化数据存储机制。

docker volume create <volume_name>: 创建一个卷。

docker volume ls: 列出所有卷。

docker volume inspect <volume_name>: 查看卷的详细信息。

docker run -v <volume_name>:<container_path>: 使用卷。

现在，即使你删除了 nginx-with-volume 容器，my_data 卷里的数据依然存在。你可以创建一个新容器，再次挂载 my_data 卷，数据就会恢复。

docker volume rm <volume_name>: 删除卷。

docker volume prune: 清理未被任何容器使用的卷。

绑定挂载 (Bind Mounts)

绑定挂载直接将主机上的一个特定目录挂载到容器中。非常适合用于开发环境，实现主机和容器之间的代码同步。

-v (pwd):/app:(pwd):/app: (pwd):/app:(pwd) 在 Linux/macOS 中代表当前终端所在的目录（Windows 下可用 %cd%）。这将主机的项目目录直接挂载到容器的 /app 目录，覆盖掉镜像中原有的 /app 内容。

现在，如果你在主机上修改 app.py 代码，容器内的 /app/app.py 也会立刻改变。配合 Flask 的 debug 模式（需在 Dockerfile 中设置 FLASK_ENV=development），可以实现代码的热重载，极大提升开发效率。

注意: 绑定挂载依赖于主机的目录结构，不利于移植。卷是 Docker 原生管理的，移植性更好。开发用绑定挂载，生产环境推荐使用卷。

网络 (Networking)

默认情况下，容器之间是网络隔离的。Docker 提供了多种网络模式来允许容器之间、容器与外部世界进行通信。

默认网络

运行 docker network ls，你会看到 Docker 自带的三个网络：

bridge: 默认网络。如果你不指定网络，容器就会连接到这个网桥。容器之间可以通过 IP 地址通信，但不能通过容器名通信。

host: 容器不会虚拟出自己的网络栈，而是直接使用主机的网络和端口。性能最好，但隔离性最差。

none: 禁用所有网络。

最佳实践与总结

最佳实践

使用 .dockerignore: 排除不需要的文件，减小构建上下文和镜像大小。

选择合适的基础镜像: 优先选择官方镜像，并选择轻量级变体（如 -slim, -alpine）。python:3.8-slim 比 python:3.8 小很多。

利用构建缓存: 将变化频率低的指令（如 COPY requirements.txt 和 RUN pip install）放在 Dockerfile 前面，变化频率高的指令（如 COPY . .）放在后面。

一个容器一个进程: 每个容器应该只关注运行一个应用或一个进程。这使得容器更容易横向扩展和调试。

不要在镜像中存储敏感数据: 不要将密码、API 密钥等写入 Dockerfile。使用环境变量 (-e 或 environment 在 Compose 中) 或 Docker Secrets（在 Swarm 中）来传入。

使用特定标签: 不要总是依赖 latest。在生产环境中使用明确的版本标签，如 my-app:1.2.3，以确保部署的一致性。

以非 root 用户运行 (安全考虑): 在 Dockerfile 中创建和使用非 root 用户。
