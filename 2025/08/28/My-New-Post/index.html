My First Post - study

2025年08月28日 1.6k 字 大概 6 分钟

核心目标：让处于不同私有网络（例如：你在家，朋友在公司或家）的设备，能够像在同一个局域网内一样相互访问（例如：通过 192.168.x.x 这样的内网 IP 直接访问对方的共享文件夹、游戏联机、远程桌面等）。

实现原理：FRP 本身是一个内网穿透工具，它的核心模式是 TCP/UDP 流量转发。我们无法真正实现一个二层网络（数据链路层），但可以通过在公网服务器上转发特定端口的流量，来模拟出局域网设备间点对点通信的效果。

FRP 服务端 (frps)：

它是什么：一个拥有公网 IP 地址的服务器（通常是租用的云服务器，如阿里云、腾讯云等）。

它的作用：作为“中介”或“交通枢纽”。所有想要组网的设备都需要先连接到它，由它来协调和转发数据流量。

核心配置：监听一个主要端口（如 7000）用于与客户端建立控制连接。

FRP 客户端 (frpc)：

它是什么：运行在你和朋友各自电脑上的程序。

它的作用：主动连接到公网上的 FRP 服务端，并告诉服务端：“我想把本机的某个端口映射到服务器上的某个端口”或者“我想访问服务器上的某个端口来找到另一台设备”。

核心配置：指定 FRP 服务端的地址和端口，并定义转发规则。
这不是真正的二层局域网：这本质上是基于端口转发的应用层代理。

详细步骤
第一步：准备公网服务器
购买或拥有一台云服务器：确保它有公网 IP（弹性公网IP也可）。系统推荐 Linux，如 Ubuntu 或 CentOS。

开放安全组/防火墙端口：登录云服务商的控制台，配置服务器的安全组规则，放行以下端口（示例端口，可自定义）：

7000：FRP 服务端与客户端的默认通信端口（必须开放）。

7001-7020：预留的一批端口，用于后续转发其他服务（如游戏联机、文件共享等）。按需开放，建议先开一段范围。

第二步：在公网服务器上安装并配置 FRP 服务端 (frps)
下载 FRP：

连接到你的服务器 (SSH)

获取最新版本下载链接

from https://github.com/fatedier/frp/releases
wget https://github.com/fatedier/frp/releases/download/v0.52.3/frp_0.52.3_linux_amd64.tar.gz
tar -xzf frp_0.52.3_linux_amd64.tar.gz
cd frp_0.52.3_linux_amd64
wget：Linux 下载命令。

tar -xzf：解压 tar.gz 压缩包。

配置服务端 (frps.ini)：使用 vi 或 nano 编辑 frps.ini 文件。

ini
[common]
bind_port = 7000
authentication_method = token
token = your_secure_token_here_123456
[common]：通用配置段。

bind_port：服务端监听的端口，客户端要连接这个端口。

token：用于身份验证的密钥，强烈建议设置，防止他人随意使用你的服务器。

authentication_method：认证方式，设置为 token。

启动 frps：

bash
./frps -c ./frps.ini
如果想在后台长期运行，推荐使用 systemd 来管理：

创建服务文件：sudo nano /etc/systemd/system/frps.service

内容如下：

ini
[Unit]
Description=Frp Server Service
After=network.target

[Service]
Type=simple
User=nobody
Restart=on-failure
RestartSec=5s
ExecStart=/path/to/your/frps -c /path/to/your/frps.ini

[Install]
WantedBy=multi-user.target
启用并启动服务：

bash
sudo systemctl enable frps
sudo systemctl start frps
sudo systemctl status frps # 检查状态
第三步：在你和朋友的电脑上安装并配置 FRP 客户端 (frpc)
下载 FRP：从同样的 GitHub Release 页面下载对应你们操作系统（Windows/macOS/Linux）的版本并解压。

配置客户端 (frpc.ini)：
这是最关键的一步。假设我们要实现：

你的设备 IP 是 192.168.1.100，你想将本机的 3389（远程桌面）端口暴露出去。

你的朋友想将他电脑上的 25565（Minecraft）端口暴露出去。

你的 frpc.ini 配置：

ini
[common]
server_addr = 1.2.3.4 # 你的公网服务器 IP
server_port = 7000 # frps 监听的端口
token = your_secure_token_here_123456 # 必须和服务器设置的一致

[rdp] # 这是一个自定义名称，代表你的远程桌面服务
type = tcp
local_ip = 192.168.1.100 # 你想暴露的本地设备的 IP
local_port = 3389 # 你想暴露的本地服务的端口
remote_port = 7001 # 在服务器上开启的端口，别人通过 server_addr:7001 访问你

[another-service] # 你可以定义更多规则
type = tcp
local_ip = 127.0.0.1
local_port = 8080
remote_port = 7002
你朋友的 frpc.ini 配置：

ini
[common]
server_addr = 1.2.3.4 # 同样的公网服务器 IP
server_port = 7000
token = your_secure_token_here_123456

[minecraft] # 一个自定义名称
type = tcp
local_ip = 192.168.2.100 # 你朋友电脑的局域网 IP
local_port = 25565
remote_port = 7003 # 在服务器上开启另一个端口，如 7003
知识点解释：

[rdp] / [minecraft]：这是一个代理规则的名称，可以任意取名，但不能重复。

type：转发协议类型，通常是 tcp 或 udp。游戏常用 udp，文件共享、远程桌面用 tcp。

local_ip / local_port：客户端所在局域网内真正提供服务的那台机器的地址和端口。

remote_port：服务端监听的端口。当有人访问 服务器IP:remote_port 时，流量会被转发到本地的 local_ip:local_port。

启动客户端：

Windows：打开命令提示符（CMD）或 PowerShell，进入 FRP 解压目录，执行：frpc.exe -c frpc.ini

macOS/Linux：终端中进入目录，执行：./frpc -c ./frpc.ini

同样可以配置为后台服务（以 Windows 为例，使用 nssm 工具）：

bash
nssm install frpc

然后在弹出的窗口中设置路径：C:\path\to\frpc.exe 和参数：-c C:\path\to\frpc.ini

nssm start frpc
第四步：连接与访问
现在，所有设备上的 FRP 客户端都应该连接到了公网服务器。

你想访问朋友的 Minecraft 服务器：
在你的游戏客户端中，输入服务器地址：1.2.3.4:7003。

流程：你的游戏数据 -> 你的电脑 -> 公网 (1.2.3.4:7003) -> FRPS -> FRPC（你朋友电脑） -> 朋友电脑上的 Minecraft 服务。

朋友想访问你的远程桌面：
他在远程桌面客户端中，输入地址：1.2.3.4:7001。

流程：朋友的 RDP 数据 -> 他的电脑 -> 公网 (1.2.3.4:7001) -> FRPS -> FRPC（你的电脑） -> 你电脑上的 RDP 服务。
