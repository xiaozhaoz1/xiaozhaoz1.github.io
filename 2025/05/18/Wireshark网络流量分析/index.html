Wireshark简介与安装

什么是Wireshark？

Wireshark是一款开源网络协议分析器，可以截取各种网络数据包，并显示数据包的详细信息。它广泛应用于网络故障排查、性能分析、网络安全检测和教育培训等领域。Wireshark支持多种操作系统（Windows、macOS、Linux），能够解析上千种网络协议，是网络专业人士必备的工具之一。

Wireshark的安装步骤

在Windows系统上安装
下载安装程序：访问Wireshark官网（https://www.wireshark.org/download.html）下载最新版本的安装程序。

以管理员身份运行安装包：右键点击安装程序，选择"以管理员身份运行"。

阅读并接受许可协议：点击"Next"继续。

选择组件：默认选择即可，第二个选项为创建桌面图标，可按照自己的情况选择。

选择安装位置：建议使用默认路径，也可自定义。

安装NPcap：安装过程中会提示安装NPcap，这是抓包所必需的驱动，务必安装。

完成安装：点击"Finish"完成安装过程。

在Ubuntu/Linux系统上安装

安装完成后，建议重启系统以确保所有驱动正确加载。

或者将当前用户添加到wireshark用户组中：

Wireshark主界面介绍

当你第一次打开Wireshark时，可能会被它的界面所震撼。别担心，我们将逐步分解各个部分的功能。

菜单栏：包含文件、编辑、视图、跳转、捕获、分析、统计、电话、工具、窗口和帮助等菜单，提供了Wireshark的所有功能入口。

工具栏：常用功能的快捷方式，如开始捕获、停止捕获、重新开始捕获等。

过滤栏：用于设置显示过滤器条件，过滤数据包列表。

数据包列表：显示捕获到的数据包，每个数据包包含编号、时间戳、源地址、目标地址、协议、长度和信息等。

数据包详细信息：显示选中数据包的所有详细信息，包括各层协议的字段和值。

数据包字节流：显示数据包对应的字节流，以十六进制和ASCII格式显示。

Wireshark的主界面中，数据包列表区的不同协议使用了不同的颜色区分。这些颜色规则可以在菜单栏的View --> Coloring Rules中查看和自定义。

捕获第一个数据包

现在让我们开始实际捕获数据包，体验Wireshark的基本功能。

选择网络接口

启动Wireshark后，主界面会显示可用的网络接口列表。

选择活跃的网络接口（通常有流量指示条）。如果是无线网络，选择WLAN或Wi-Fi接口；如果是有线网络，选择以太网接口。

点击选中的网络接口即可开始捕获数据包。

开始捕获数据

点击选中的网络接口后，Wireshark立即开始捕获数据包。

打开浏览器，访问任何一个网站，如http://www.google.com，生成一些网络流量。

回到Wireshark，你会看到数据包列表在不断更新。

捕获到一定数量的数据包后（20-30个），点击工具栏中的红色方形按钮停止捕获。

保存捕获文件

捕获完成后，建议保存数据包文件以便后续分析：

点击菜单栏的 File > Save。

选择保存位置和文件名。

点击保存按钮。

Wireshark默认保存为.pcapng格式，这是推荐的保存格式，因为它包含了更多的元信息。你也可以选择保存为传统的.pcap格式，以兼容一些老版本工具。

Wireshark过滤器使用技巧

Wireshark的强大功能之一就是过滤器系统，它可以帮助我们从海量的数据包中筛选出我们感兴趣的内容。Wireshark提供了两种类型的过滤器：捕获过滤器和显示过滤器。

捕获过滤器

捕获过滤器在开始捕获前设置，用于限制只捕获符合条件的数据包，可以减少捕获的数据量，提高效率。

基本语法
捕获过滤器使用伯克利封包过滤器（BPF） 语法，由四个核心元素组成：

类型（Type）：主机（host）、网段（net）、端口（port）

方向（Dir）：源地址（src）、目标地址（dst）

协议（Proto）：tcp、udp、http、icmp等

逻辑运算符：与（&&）、或（||）、非（！）

常用捕获过滤器示例

设置捕获过滤器的方法：

点击Capture > Options。

在"Capture Filter"输入框中输入过滤条件。

输入框变绿色表示语法正确，红色表示语法错误。

点击Start开始捕获。

显示过滤器

显示过滤器用于在捕获完成后筛选数据包，不会删除数据，只是隐藏不符合条件的数据包。

比较运算符
显示过滤器支持多种比较运算符：

==：等于

!=：不等于

：大于

<：小于

=：大于等于

<=：小于等于

contains：包含

matches：匹配正则表达式

逻辑运算符
and：逻辑与

or：逻辑或

not：逻辑非

in：包含于

常用显示过滤器示例

高级过滤技巧

除了基本过滤外，Wireshark还提供了一些高级过滤功能：

基于数据包大小的过滤

基于特定内容的过滤

基于时间间隔的过滤

基于TCP标志位的过滤

保存和管理过滤器

如果你经常使用某些过滤器，可以将它们保存为过滤器按钮或过滤器表达式：

在过滤器输入框输入过滤条件

点击过滤器输入框右侧的+按钮

输入过滤器名称

点击OK保存

这样下次使用时，只需点击已保存的过滤器按钮即可应用该过滤条件。

分析常见协议

接下来，我们学习如何使用Wireshark分析常见的网络协议。

分析TCP协议

TCP（传输控制协议）是互联网中最常用的传输层协议，提供可靠的、面向连接的通信。

TCP三次握手分析
TCP通过三次握手建立连接：

SYN：客户端发送SYN包（SYN=1, ACK=0）到服务器，序列号为随机值（如Seq=100）

SYN-ACK：服务器回应SYN-ACK包（SYN=1, ACK=1），序列号为随机值（如Seq=300），确认号为客户端序列号加1（Ack=101）

ACK：客户端发送ACK包（SYN=0, ACK=1），序列号为101，确认号为服务器序列号加1（Ack=301）

使用过滤器查看TCP三次握手：

TCP数据传输
建立连接后，双方开始数据传输。每个TCP包都包含序列号和确认号，确保数据按序到达和确认。

TCP四次挥手
TCP通过四次挥手终止连接：

FIN：一方发送FIN包（FIN=1, ACK=1）

ACK：另一方回应ACK包

FIN：另一方发送FIN包

ACK：一方回应ACK包

分析HTTP协议

HTTP是应用最广泛的Web协议，Wireshark可以很好地解析和分析HTTP流量。

HTTP请求和响应

分析HTTP头信息
在数据包详细信息面板，可以查看HTTP请求和响应的详细信息：

Request Method：请求方法（GET、POST等）

Request URI：请求的URI

Request Version：HTTP版本

Host：主机名

User-Agent：客户端浏览器信息

Response Code：响应状态码

Content-Type：内容类型

提取HTTP传输的文件
过滤HTTP流量：http

选择文件传输的HTTP响应包

右键选择Follow > HTTP Stream

在打开的窗口中选择Save as保存文件

分析DNS协议

DNS（域名系统）将域名解析为IP地址，是互联网的重要基础设施。

DNS数据包中的重要字段：

Transaction ID：事务ID，匹配查询和响应

Queries：查询部分，包含查询名称、类型和类

Answers：响应部分，包含资源记录

分析ICMP协议

ICMP（互联网控制消息协议）用于网络设备之间发送控制消息。

网络故障排查实战

Wireshark是网络故障排查的强大工具，下面介绍一些常见问题的排查方法。

网络延迟问题

当网络访问缓慢时，可以使用Wireshark分析延迟原因。

分析方法：

添加时间列：右键列标题 > Column Preferences > 添加列 “Time since previous frame”

如果连续包间隔 >200ms，可能存在网络延迟

检查是否有大量重传包，表明网络不稳定

检查是否有零窗口包，表明接收方处理能力不足

数据丢包问题

分析方法：

打开Statistics > TCP Stream Graph > Time-Sequence Graph

观察序列号增长情况，突然下降表示重传

重传率超过5%可能影响网络性能

连接异常问题

分析方法：

检查RST包来源，可能是服务端主动拒绝连接

检查ICMP不可达消息，可能是路由问题或防火墙拦截

DNS问题

分析方法：

检查DNS响应代码：0表示成功，3表示域名不存在，其他值表示不同类型的错误

比较DNS查询和响应时间，长时间无响应可能表示DNS服务器问题

网络安全分析

Wireshark也是网络安全分析的重要工具，可以检测各种网络攻击和异常行为。

检测端口扫描

端口扫描是攻击者收集目标信息的常用技术。

检测方法：

应用过滤器查看SYN包

如果发现从单个源IP到多个目标端口的大量SYN包，可能是端口扫描

使用Statistics > Conversations > TCP查看对话统计

检测DoS攻击

拒绝服务（DoS）攻击试图耗尽目标资源使其无法提供服务。

检测方法：

统计特定类型数据包的出现频率

检查是否来自大量源IP向单一目标发送流量

使用Statistics > IO Graphs查看流量趋势

检测可疑DNS活动

攻击者可能使用DNS隧道泄露数据或进行命令控制。

检测方法：

查找异常长的域名或对于同一域的大量DNS请求

检查非标准记录类型（如TXT）的查询

查找域名中包含可疑编码（如Base64）的查询

识别明文凭据传输

检测方法：

应用上述过滤器查找敏感信息

检查HTTP登录表单提交的POST请求

关注使用HTTP而不是HTTPS的登录页面

进阶技巧与最佳实践

使用命令行工具

Wireshark提供了命令行工具tshark和dumpcap，可用于自动化捕获和分析。

使用dumpcap捕获数据

使用tshark分析数据

配置个性化解码器

如果Wireshark不能正确解析某些自定义协议，你可以添加解码器：

点击Analyze > Decode As

选择字段和值

选择要解码为的协议

点击OK保存设置

使用图表和图形

Wireshark提供了多种统计图表，帮助可视化分析流量：

IO Graphs：显示流量随时间变化趋势

Round Trip Time Graph：显示TCP往返时间

Throughput Graph：显示吞吐量

TCP Stream Graph：显示TCP流序列号随时间变化

重组网络会话

Wireshark可以重组TCP流，查看完整会话：

选择TCP包

右键选择Follow > TCP Stream

查看完整的请求和响应内容

可以保存整个会话内容

最佳实践

明确分析目标：抓包前明确要解决的问题，避免捕获过多无用数据

选择合适的抓包位置：根据问题选择在客户端、服务端或中间节点抓包

使用过滤器：尽量使用捕获过滤器减少数据量，显示过滤器聚焦分析重点

保存基线数据：在正常运行时保存基线数据，便于与异常时对比

注意隐私和法律：不要在不授权的网络抓包，注意保护敏感数据

分段捕获大流量：对于高流量网络，使用循环缓冲区或多文件捕获

文档化分析过程：记录分析步骤和发现，便于后续回顾和分享

总结

Wireshark是一个功能强大的网络分析工具，学习曲线可能有些陡峭，但掌握它将为你提供前所未有的网络洞察能力。本文从基础安装到高级技巧，提供了全面的Wireshark使用指南，希望能够帮助你开始使用这个强大工具。

记住，成为Wireshark专家需要时间和实践，建议从简单的情境开始，逐步尝试更复杂的分析任务。网络抓包和分析是一门实践性很强的技能，只有通过实际操作才能真正掌握。
